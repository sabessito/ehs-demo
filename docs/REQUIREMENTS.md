# 요구사항 체크리스트 — 스마트 안전기술 기획 면접용 앱

> 하영우님이 대화 전체에서 요청한 사항을 누적 정리. 각 항목의 이행 여부는
> "제작자(Builder)"가 구현하고 "검토자(Reviewer)"가 독립 검증한다.

## ★ 상시 제약 (모든 작업에 항상 적용)
- [G1] **모든 개발 업무는 데스크탑/노트북 없이 '폰'으로만 수행한다.**
       (git, 배포, 코드 수정, 확인까지 전부 모바일 기준으로 안내·설계할 것)
- [G2] **Claude는 작업을 시작하기 전 항상 이 요구사항 파일을 먼저 확인하고,
       반영했는지 점검한 뒤 실행한다.** (제작자/검토자 교차검증 유지)

## A. 앱 기본 (1차 요청)
- [A1] 모바일(폰)에서 보는 소개용 앱
- [A2] EHS 전략 Full Process를 단계별로 보여줄 것:
       데이터 분석 → 이슈 발굴 → 전략 → 핵심 과제(기획) → 구축/로드맵
- [A3] 인터넷/중대재해 법규 기반의 샘플 데이터 활용
- [A4] 계열사(협력 기업) 전파/컨설팅 관점이 드러날 것

## B. 데이터 분석 3대 관점 (2~3차 요청)
- [B1] 데이터를 3가지 큰 관점(오브젝트)으로 표현
- [B2] 오브젝트 = Cost(예방투자) / Risk Index(위험도 지수) / Limit(운영기준선)
- [B3] "기업은 이윤 추구 → 적정선 줄타기" 관점 반영
       (비용 대비 위험, 법적 하한선 동시 검증)
- [B4] ALARP 프레임워크를 음영(영역)으로 시각화
- [B5] 법적 최소 기준선을 별도 표시(중대재해법은 비용 과다 항변 불인정)

## C. 실제/정부 데이터 (4~5차 요청)
- [C1] 샘플 데이터가 "무엇인지" 명확히(출처/대상 명시)
- [C2] 특정 기업·특정 영역 "전체"를 보여주듯 단일 사업장 케이스화
- [C3] 한국 기준: 고용노동부 2024 재해유형 분포 반영
- [C4] 큰 그림: 전체 노동자 수 / 사망자 수 / 부상(사고재해자) 수 함께 표시
- [C5] 해외 사업장 케이스 별도 확보(비중 항목이 한국과 다름)
- [C6] 해외 케이스도 한국과 "동일 기준"으로 정리
- [C7] 해외 케이스 중 자국법 위반/중대재해법 상당 상황·사고 확인
- [C8] 앱에서 한국/해외 케이스를 "선택 가능"하게 토글 구현

## D. 프로세스 (5차 요청)
- [D1] 요구사항 리스트 파일 작성(이 파일)
- [D2] 제작자/검토자 두 역할로 교차 검증, 더 나은 결과 제시

## E. 품질 가드레일 (상시)
- [E1] 출처 명시 / 가상 데이터와 공식 통계 구분
- [E2] 모바일 가독성(420px, 한 화면 단위)
- [E3] 코드가 실제로 컴파일/렌더 가능(괄호 균형, recharts/SVG 정상)

### 검토자가 잡은 한계 (정직한 고지)
- Cost·Limit·Event 가중치는 내부 운영 데이터 성격이라 전부 가상값. 면접 시 "구조 시연용"임을 명시할 것.
- 해외 비치명 부상 약 260만건은 추정치(민간 전체). 정확 수치는 BLS SOII 별도 확인 필요.
- 실제 마이크로데이터(KOSHA 13.8만건)로 업종 필터 집계를 원하면 XLSX 업로드 시 반영 가능.
