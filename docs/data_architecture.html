데이터 아키텍처

원본의 최소 단위는 재해자/사건 레코드입니다. 이를 집계(GROUP BY)해 비중%이 나오고, 그 비중이 가공·산출을 거쳐 앱의 최종 데이터셋이 됩니다.

▸ 각 항목을 탭하면 실제 데이터셋 샘플이 펼쳐집니다 (레코드 → 집계 → 비중 순서)

④ 최종 — 앱 탑재 DATASETS

DATASETS ├─ korea │ ├─ bigPicture ← KR-2 │ ├─ hazardShare ← KR-1 │ ├─ cost ← IN-1 (가상) │ ├─ riskIndex ← KR-1 + IN-2 │ ├─ alarp ← cost × index │ └─ issues ← 중대재해법 └─ overseas ├─ bigPicture ← US-1 ├─ hazardShare ← US-2 ├─ cost/riskIndex/alarp └─ issues ← OSHA
공식 통계 내부 가상값
핵심: 공개 가능한 분포·규모는 정부 통계 그대로, 운영 비용·기준선만 가상으로 — "공개 데이터와 영업비밀을 구분해 설계할 줄 안다"는 점을 데이터 계보로 증명.
